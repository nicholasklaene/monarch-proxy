# monarch-bridge-client
HTTP/JSON wrapper over Monarch Money's GraphQL API.

**Contract conventions:**
- Every successful 2xx body is `{\"data\": <typed object>}` (except `/healthz`
  and `/v1/auth/{status,refresh}`, which return plain bodies so health probes
  stay zero-parse).
- Failure bodies, when returned, are `{\"code\": ..., \"message\": ..., \"details\": {...}}`.
- Mutations whose application layer reports errors come back as `HTTP 422`
  (the proxy walks Monarch's `data.<mutation>.errors[]` and surfaces it).
- No per-request auth on the wire - the bridge holds Monarch's session token
  itself, loaded once via the interactive bootstrap CLI.

**Source of truth.** This multi-file YAML tree is hand-curated and
authoritative. Both request DTOs and response DTOs are generated from it via
openapi-generator-gradle-plugin into `build/generated/openapi/`, along with
a Spring `*Api` interface per tag that controllers implement directly. The
spec drives the web layer end-to-end.
