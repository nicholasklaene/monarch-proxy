# Sample generated clients

This directory holds OpenAPI-generated clients in three languages, committed as a showcase of what consumers get for free from `openapi/monarch-bridge.yaml`. None of this code is hand-edited. Treat it as a reference, not a dependency.

The per-language `README.md` files in each subdirectory are themselves generated by openapi-generator and document every endpoint, request DTO, and response DTO. They are excellent reference material even if you do not plan to use the generated client.

## What is here

| Path | Generator | Source lines | Disk |
|---|---|---|---|
| `clients/python/` | `python` (urllib3) | ~71k | ~7.3M |
| `clients/typescript/` | `typescript-axios` | ~18k (single `api.ts`) | ~2.7M |
| `clients/kotlin/` | `kotlin` (default OkHttp + Gson) | ~28k | ~5.9M |
| `clients/bash/` | `bash` (single `monarch-api` script + completions) | ~20k (single script) | ~1.1M |

Each tree includes a `docs/` folder with one Markdown file per operation and one per schema (~400 files per language). Those are also generator output.

The bash client requires Bash 4.3+ (macOS ships 3.2; `brew install bash`). The curated `cli/monarch` shell wrapper is bash-3.2 compatible and remains the daily driver; the generated client is the parity safety net for any operation `cli/monarch` doesn't wrap.

## Regenerating

The spec at `openapi/monarch-bridge.yaml` is authoritative. To refresh any client after a spec change:

```bash
# Python
npx @openapitools/openapi-generator-cli generate \
  -i openapi/monarch-bridge.yaml \
  -g python \
  -o clients/python \
  --additional-properties packageName=monarch_bridge_client

# TypeScript (axios)
npx @openapitools/openapi-generator-cli generate \
  -i openapi/monarch-bridge.yaml \
  -g typescript-axios \
  -o clients/typescript \
  --additional-properties npmName=monarch-bridge-client

# Kotlin
npx @openapitools/openapi-generator-cli generate \
  -i openapi/monarch-bridge.yaml \
  -g kotlin \
  -o clients/kotlin \
  --additional-properties packageName=com.nicholasklaene.monarchbridge.client

# Bash (pinned to 2.20.5; the bash generator was removed in 7.x)
npx @openapitools/openapi-generator-cli@2.20.5 generate \
  -i openapi/monarch-bridge.yaml \
  -g bash \
  -o clients/bash \
  --additional-properties scriptName=monarch-api
```

Build outputs (`node_modules/`, `dist/`, `build/`, `.gradle/`, `*.egg-info/`, virtualenvs) are gitignored per language. Only generator output is checked in.

## Generating a subset client

If you only need a few tags' worth of endpoints, append `--global-property apis=Tag1,Tag2`. See the "Generate a subset client" section in the top-level `README.md` for the full tag list and common subset recipes.

## Why these four

These were picked to cover the most common consumer environments:
- **Python** for scripting and notebooks.
- **TypeScript** for browser and Node clients.
- **Kotlin** for JVM consumers and Android.
- **Bash** for full shell parity (one subcommand per operationId), as a safety net for any endpoint `cli/monarch` doesn't wrap. See [bash/README.md](bash/README.md).

The OpenAPI generator supports 50+ targets. Run `npx @openapitools/openapi-generator-cli list` for the full menu; Go, Rust, Swift, Java, PHP, C#, and Ruby are all one command away with the same spec.
